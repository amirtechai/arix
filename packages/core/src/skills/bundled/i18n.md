---
description: Internationalisation — string extraction, locale management, RTL, ICU
---

You add or maintain i18n. Externalise every user-visible string; never concatenate translated fragments.

Workflow:
1. **Audit** — grep for hardcoded user-visible strings in JSX/templates/UI code.
2. **Extract** — replace with `t('namespace.key')` and add to default locale file.
3. **Pluralisation** — use ICU/CLDR plural rules; never `count + ' items'`.
4. **Interpolation** — variables stay as named placeholders, not positional.
5. **Date/number/currency** — `Intl.*` APIs; never hand-format.
6. **RTL** — logical CSS properties (`margin-inline-start`), `dir="rtl"` testing.
7. **Bidi** — wrap LTR content inside RTL with Unicode bidi marks where needed.

File hygiene:
- One key per logical message, namespaced by feature.
- Comments/context for translators on ambiguous keys.
- No HTML in messages — use rich-text components with safe slot interpolation.
- Lock keys to source language; flag missing translations in CI.

Tools: i18next, FormatJS, Vue I18n, react-intl, gettext, Crowdin/Lokalise/Phrase for TMS.

Anti-patterns: string concatenation across locales, gendered grammar baked into English-only logic, time-zone–naive timestamps shown to users.
